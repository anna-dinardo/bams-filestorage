Overview

SharePoint is our #1 Recommended cloud service provider for good reason. With countless easy-to-use tools, SharePoint makes collaboration and file storage more efficient and customizable. To get the most out of this platform, BAMS strongly urges utilizing features like Workflow, Customizable Views, Version History, and Metadata.

Workflows

Send your documents through automated workflows that inform your colleagues when it's time to provide input or ask your manager to review and approve your stuff.

What are SharePoint Workflows?

A SharePoint workflow is like a flowchart that operates automatically requiring a lot less work.

To illustrate this, we will use the example above. Doing this process manually can often mean time spent waiting around for documents to be shared, check, approved, and tracked. This also can lead to different versions of the same document which leads to confusion and disorganization.

Using the SharePoint document approval workflow, all of the checking up, reminders, and sharing is done automatically. Notifications are sent out when someone is late to completing their part, and the workflow is proactively monitoring each step.

There are four commonly used Workflows that can be set up within SharePoint: Approval, Collect Feedback, Collect Signatures, and the Three-state Workflow.

1. Approval

The Approval Workflow is used to share a document or an item in a SharePoint list or library with one or more individuals for approval.

Once the approval workflow start, it assigns a task to each specified individual. They can approve or reject the item, request a change, or cancel the task.

2. Collect Feedback

The Collect Feedback Workflow is used to gather feedback on a document. The workflow set-up is similar to the approval workflow and you can add multiple instances of the collect feedback workflow to a site.

When adding this workflow, you need to consider how the feedback will be collected:

- Leave comments on the task form only.

- Insert track changes and add comments to the item itself.

This workflow must be activated by the site administrator at the site collection level.

3. Collect Signatures

The Collect Signatures Workflow is used with Word documents, Excel workbooks, and InfoPath forms.

Before starting the workflow, you must add signature lines for each required individual. Once a signature is added to the signature line, the document is locked from all changes aside from adding more signatures.

4. Three-state workflow

The Three-state Workflow tracks the status of a list item, like a task, through three states or phases. The workflow can be used with any list that has a choice column with three or more values, for example, not started, in progress, and done.

Before starting the workflow, you must add signature lines for each required individual. Once a signature is added to the signature line, the document is locked from all changes aside from adding more signatures.

To add or start a workflow, you must have the correct permission for the list, library, or site collection where the workflow runs:

To add a workflow   By default, you must have the Manage Lists permission to add a workflow. (The Owners group has the Manage Lists permission by default; the Members group and the Visitors group do not.)

To start a workflow  Also by default, you must have the Edit Items permission to start a workflow thatâ€™s already been added. (The Members group and the Owners group both have the Edit Items permission by default; the Visitors group does not.)

Alternatively, Owners can choose to configure specific workflows so that they can be started only by members of the Owners group.

Metadata

Metadata is one of SharePoint's most powerful features, and used correctly, can be one the greatest tools in keeping files organized and quickly accessbile. It is information information about informationA book's title and author or a catalog's number are metadata. When you use SharePoint metadata, you are adding information to your files to make them easier to find.Understanding what metadata means is easy. The part to using metadata is planning it out and sticking to it.

What is metadata?

The key part of metadata is that is describes information about an object, giving context and detail about a topic. Take a car for example, How would you describe it? It has a make, model, year, color, etc. These properties of the car that give it context and value is what we call metadata.

With SharePoint metadata, some fields are auto-populated and others are there for you to add input. In the most basic document library, metadata is typically limited to file type (icon), file name, modified date, and modified by. But there are actually numerous other fields available to you to choose from. You can see the full list of options below.

If you are the site owner, you can create metadata by going to Library > + Add Column.

As long as you have Contribute access within the library, you can edit the metadata for individual files by selecting ... (ellipses) > Details. In the side panel, scroll down to properties and select Edit all.

To add one of the pre-existing columns SharePoint offers (shown previously), go to (Your SharePoint Library) > All Documents > Edit Current View.

Custom Views

SharePoint Views are the basic structure of your site. Lists and Libraries have views that organize and group your data.

There are two options when editing views

  • Personal - visible to you only

  • Public - visible to everyone (must have permission)

To create anew view in a SharePoint Library go to Library > Settings > Library Settings On the bottom of the Library settings page, select Create View.

Version History

Likely one of the most useful tools within Sharepoint, version history keeps a running list of changes that occured during the lifetime of your file. Version history lets you access old versions and shows you the quanitity of each version, who created it, and the size of the file.

Why you should be using version history

Sharepoint's version history creates a new file:

  • - every time you save youre file

  • - every time you check in a file

  • - when you overwrite an existing file

  • - and if youâ€™re co-authoring, every half hour after someone starts editing a file.

It is important that you make sure version history is enabled for you library, dont just assume!. To enable or check version history:

  • 1. Go to the library or list for which you want to enable

  • 2. Select Settingsand then select Library Settings or List Settings

  • 3. On the settings page, select Versioning Settings

To view or restore an earlier version of a document, follow these steps:

  • 1. Go to the list or library with the item or document for which you want to view history.

  • 2. Next to the item for which you want to view history, select the ... (ellipses).

  • 3. Select Version History
