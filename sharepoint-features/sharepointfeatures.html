Overview

SharePoint is our #1 Recommended cloud service provider for good reason. With countless easy-to-use tools, SharePoint makes collaboration and file storage more efficient and customizable. To get the most out of this platform, BAMS strongly urges utilizing features like Workflow, Customizable Views, Version History, and Metadata.

1. Metadata

Metadata is one of SharePoint's most powerful features, and used correctly, can be one the greatest tools in keeping files organized and quickly accessbile. It is information information about informationA book's title and author or a catalog's number are metadata. When you use SharePoint metadata, you are adding information to your files to make them easier to find. Understanding what metadata means is easy. The part to using metadata is planning it out and sticking to it.

What is metadata?

The key part of metadata is that is describes information about an object, giving context and detail about a topic. Take a car for example, How would you describe it? It has a make, model, year, color, etc. These properties of the car that give it context and value is what we call metadata.

With SharePoint metadata, some fields are auto-populated and others are there for you to add input. In the most basic document library, metadata is typically limited to file type (icon), file name, modified date, and modified by. But there are actually numerous other fields available to you to choose from. You can see the full list of options below.

If you would like to add metadata to a list or library, please contact BAMS.

2. Sorting and Filtering

Sorting your list and libraries, and implementing metadata go hand in hand. The more your team uses SharePoint libraries and lists, the more they will fill up. Eventually there will be pages of information, making it difficult for you to find what you are looking for. If you know what your are looking for, you can search for it (read more about searching below). However, if you dont know the name of the file, You can use column heading, and your metadata to sort the list.

This list is in alphabetical order by file name, with folders being displayed first. You can change this by selecting the column heading and choose how you wish for that column to be sorted. In the example below, the list can be filtered alphabetically by first name.

4. Searching

Searching in SharePoint is more than typing a few keywords and pressing enter, it has a ton of additional features that allow you to enhance the quality of your search query and get significantly better search results.

Where to Search in SharePoint

It's important to know where youre searching. You can get more precise results just by searching in a particular library as opposed to the entire site.

Sitewide search boxes lets search the site youâ€™re in plus all sub-sites below that site, which means that your results might be more wide-ranging than you might have intended.

You can also search a library which will return results that live directly in the library. The search bar at the top of SharePoint will search locations based on where you are in the site. Searching on the homepage will search the entire site, while searching in a document library will search only the contents of that library.

5. Version History

Likely one of the most useful tools within Sharepoint, version history keeps a running list of changes that occured during the lifetime of your file. For example, you don't need to save a new file with a date or version number in the title. Version history lets you access old versions and shows you the quanitity of each version, who created it, and the size of the file.

Why you should be using version history

Sharepoint's version history creates a new file:

  • - every time you save youre file

  • - every time you check in a file

  • - when you overwrite an existing file

  • - and if youâ€™re co-authoring, every half hour after someone starts editing a file.

It is important that you make sure version history is enabled for you library, dont just assume! . If you donâ€™t have permission, contact BAMS. To enable or check version history:

  • 1. Go to the library or list for which you want to enable

  • 2. Select Settingsand then select Library Settings or List Settings

  • 3. On the settings page, select Versioning Settings

To view or restore an earlier version of a document, follow these steps:

  • 1. Go to the list or library with the item or document for which you want to view history.

  • 2. Next to the item for which you want to view history, select the ... (ellipses).

  • 3. Select Version History
